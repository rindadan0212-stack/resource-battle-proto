# リソース制バトル — 戦闘ロジック検証プロトタイプ

ターン制1対1RPGの戦闘システム(設計 v2)を、ブラウザ上で**実プレイ＋バランス検証**するための自己完結HTML(依存なし)。
アート・シナリオ等は含まない、**戦闘ロジックだけ**のサンドボックスです。数値は検証用の仮値。

▶ **プレイ:** `index.html` をブラウザで開く(または GitHub Pages)。

## 設計の要点
- **1手番 = エネルギー(行動力)の予算内で複数行動**を組み立てる(シャドバのPP感覚)。
- 行動は **攻撃 / スキル(強撃・崩し・見切り) / 防御 / どうぐ** の4種。
- **バフ・回復は「どうぐ」のみ**(消耗品=お金／発動型淫具=消耗なしだが使用でEP上昇)。
- **興奮(EP)が上がるほど最大エネルギーが減少** → 手数が減り、100で**絶頂敗北**。
  「感じさせられながら押し切られる」死のスパイラルが戦闘の芯。
- **武具/防具**が最大エネルギー±・特殊効果を持ち、ビルドが変わる。

## 「殴り合い脱却」の数値証明
ページ下部の **オート対戦** は、同条件で **攻撃連打AI** と **賢いAI**(EPレース管理・大技を見切り)を各N戦し、勝率を比較します。
**賢い ＞ 連打** なら、リソース管理に意味がある＝単なる殴り合いを脱している、ということ。

検証例(各1000戦):

| 相手 | 攻撃連打AI | 賢いAI | 差 |
|---|---|---|---|
| ボス雑魚(EPレース) | 43% | 64% | **+21** |
| ボス・軽装(大技の脅威) | 79% | 100% | **+21** |
| ボス・重装 | 93% | 98% | +5 |

## 位置づけ
本HTMLで数値を詰めたのち、確定値をゲーム本体(Ren'Py)の戦闘エンジンへ移植する予定の**プロトタイプ**です。
